# Provision ERD

아래 ERD는 현재 [04_create_provision_tables.sql](D:\sts-5.1.1.RELEASE\workspace\hi-locf\src\main\resources\db\oracle\04_create_provision_tables.sql) 기준의 충당금 테이블 구조를 Mermaid 형식으로 표현한 것입니다.

- 공통 원천 테이블
  - `CUSTOMER_MASTER`
  - `LOAN_CONTRACT`
- 충당금 보강 원천 테이블
  - `CUSTOMER_RATING`
  - `LOAN_DELINQUENCY`
  - `LOAN_COLLATERAL`
- 충당금 배치/중간/결과 테이블
  - `PRV_BATCH_EXECUTION`
  - `PRV_BATCH_STEP_EXECUTION`
  - `PRV_TARGET_CONTRACT`
  - `PRV_STAGE_RESULT`
  - `PRV_EAD_RESULT`
  - `PRV_PD_RESULT`
  - `PRV_LGD_RESULT`
  - `PRV_ECL_RESULT_DTL`
  - `PRV_ECL_RESULT_SUMMARY`

```mermaid
erDiagram
    CUSTOMER_MASTER {
        NUMBER CUSTOMER_ID PK
        VARCHAR CUSTOMER_NO
        VARCHAR CUSTOMER_NAME
        VARCHAR CUSTOMER_TYPE
        VARCHAR SEGMENT_CODE
        VARCHAR STATUS_CODE
    }

    LOAN_CONTRACT {
        NUMBER CONTRACT_ID PK
        VARCHAR CONTRACT_NO
        NUMBER CUSTOMER_ID FK
        VARCHAR PRODUCT_CODE
        VARCHAR LOAN_TYPE_CODE
        VARCHAR REPAYMENT_TYPE
        DATE EXECUTION_DATE
        DATE MATURITY_DATE
        NUMBER PRINCIPAL_AMT
        VARCHAR STATUS_CODE
    }

    CUSTOMER_RATING {
        NUMBER RATING_ID PK
        NUMBER CUSTOMER_ID FK
        DATE BASE_DATE
        VARCHAR RATING_GRADE

    }

    LOAN_DELINQUENCY {
        NUMBER DELINQUENCY_ID PK
        NUMBER CONTRACT_ID FK
        DATE BASE_DATE
        NUMBER DELINQUENCY_DAYS
        VARCHAR DEFAULT_YN
    }

    LOAN_COLLATERAL {
        NUMBER COLLATERAL_ID PK
        NUMBER CONTRACT_ID FK
        DATE BASE_DATE
        VARCHAR COLLATERAL_TYPE
        NUMBER COLLATERAL_VALUE_AMT
        NUMBER RECOVERY_RATE
    }

    PRV_BATCH_EXECUTION {
        NUMBER BATCH_EXECUTION_ID PK
        VARCHAR BATCH_RUN_NO

        VARCHAR BATCH_TYPE
        VARCHAR STATUS_CODE

        NUMBER ERROR_COUNT
        TIMESTAMP STARTED_AT
        TIMESTAMP FINISHED_AT
    }

    PRV_BATCH_STEP_EXECUTION {
        NUMBER STEP_EXECUTION_ID PK
        NUMBER BATCH_EXECUTION_ID FK

    PRV_TARGET_CONTRACT {

        NUMBER DELINQUENCY_DAYS

        NUMBER RECOVERY_RATE

    PRV_STAGE_RESULT {

    PRV_EAD_RESULT {
        NUMBER EAD_RESULT_ID PK
        NUMBER BATCH_EXECUTION_ID FK
        NUMBER TARGET_ID
        NUMBER CONTRACT_ID
        VARCHAR CONTRACT_NO
        NUMBER CARRYING_AMOUNT
        NUMBER EAD_AMOUNT
    }

    PRV_PD_RESULT {
        NUMBER PD_RESULT_ID PK
        NUMBER BATCH_EXECUTION_ID FK
        NUMBER TARGET_ID
        NUMBER CONTRACT_ID
        VARCHAR CONTRACT_NO
        VARCHAR STAGE_CODE
        NUMBER ONE_YEAR_PD_RATE
        NUMBER LIFETIME_PD_RATE
    }

    PRV_LGD_RESULT {
        NUMBER LGD_RESULT_ID PK
        NUMBER BATCH_EXECUTION_ID FK
        NUMBER TARGET_ID
        NUMBER CONTRACT_ID
        VARCHAR CONTRACT_NO
        NUMBER RECOVERY_RATE
        NUMBER LGD_RATE
    }

    PRV_ECL_RESULT_DTL {
        NUMBER RESULT_DTL_ID PK
        NUMBER BATCH_EXECUTION_ID FK
        DATE BASE_DATE
        NUMBER CONTRACT_ID
        VARCHAR CONTRACT_NO
        VARCHAR CUSTOMER_NAME
        VARCHAR PRODUCT_CODE
        VARCHAR STAGE_CODE
        VARCHAR STAGE_REASON

        NUMBER EAD_AMOUNT

        NUMBER LGD_RATE
```

## 읽는 기준

- `CUSTOMER_MASTER`, `LOAN_CONTRACT`
  - 공통 운영계 원천
- `CUSTOMER_RATING`, `LOAN_DELINQUENCY`, `LOAN_COLLATERAL`
  - 충당금 산출에 필요한 보강 원천
- `PRV_BATCH_EXECUTION`
  - 충당금 배치 실행 1건 헤더
- `PRV_BATCH_STEP_EXECUTION`
  - Stage/EAD/PD/LGD/ECL 단계별 실행 이력
- `PRV_TARGET_CONTRACT`
  - 기준일자에 실제 충당금 산출 대상이 된 계약
- `PRV_STAGE_RESULT`
  - 계약별 Stage 판정 결과
- `PRV_EAD_RESULT`
  - 계약별 EAD 산출 결과
- `PRV_PD_RESULT`
  - 계약별 1년/평생 PD 결과
- `PRV_LGD_RESULT`
  - 계약별 회수율/LGD 결과
- `PRV_ECL_RESULT_DTL`
  - 계약별 최종 ECL 결과
- `PRV_ECL_RESULT_SUMMARY`
  - 기준일자/상품/Stage 단위 요약 결과
